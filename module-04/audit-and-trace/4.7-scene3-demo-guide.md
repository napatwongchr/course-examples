## Part 1 — แสดงปัญหา

**Instruction ที่ใช้:** V1 Basic (ใส่ไว้ก่อนแล้ว)

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Claude Desktop → Project "Task Planning" | — |
| 2 | กด New Chat → แนบไฟล์ `prd-demo.md` → Enter | "ลองรัน Workflow กับ PRD ตัวนี้ดูครับ" |
| 3 | รอ response → เลื่อนดู Task List | "ได้ Task List ออกมา ดูเหมือนจะโอเคนะครับ" |
| 4 | **ชี้ที่หน้าจอ** → ไม่มี Task ด้าน Testing เลย | "แต่รอก่อนนะครับ — Task ด้าน Testing ไม่มีเลยสักข้อ" |
| 5 | หยุดนิ่ง 2 วินาที | "แล้วเราจะรู้ได้ยังไงว่าทำไม?" |
| 6 | ชี้ที่ response ทั้งหน้า | "ไม่มีข้อมูลใดในนี้ที่บอกเราได้เลย" |

## Part 2 — เพิ่ม Auditability

**เป้าหมาย:** แสดงให้เห็น Audit Block ที่ท้าย response

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | คลิก **Project Instructions** → Edit | "เราจะแก้ Instruction นิดนึงครับ" |
| 2 | **ลบ** Instruction V1 ออก → วาง V2 ลงไป เราจะเพิ่มส่วนของ audit block ที่บอกว่า input มาจากไหน และ task แตกจากอะไร | "เพิ่มส่วนที่ให้ AI บันทึก Audit Block ท้าย response" |
| 3 | Save → เปิด New Chat → แนบ `prd-demo.md` ใหม่ | — |
| 4 | รอ response → **เลื่อนลงท้ายสุด** | "ลองดูที่ท้าย response ครับ" |
| 5 | **ชี้ที่ Audit Block** | "ตอนนี้มี Audit Block ปรากฏขึ้นมาแล้ว" |
| 6 | อ่านแต่ละบรรทัดใน Audit Block ให้ผู้ชมฟัง | "Input คืออะไร — ทำเมื่อไหร่ — ใช้ Rule อะไร" |

> 💡 **จุดที่ต้องเน้น:** "ถ้าวันหลังมีปัญหา เปิด conversation history มาดู Audit Block ได้เลย ไม่ต้องเดา"

## Part 3 — เพิ่ม Traceability + Aha Moment

**เป้าหมาย:** แสดง Task ที่มีเส้นเชื่อมกลับ PRD → แล้วพิสูจน์ว่า Testing หายไปเพราะ PRD

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เลื่อนขึ้นไปดู Task table | "แต่ยังมีอีกอย่างครับ — ดูที่ Task table" |
| 2 | **ชี้คอลัมน์ "มาจาก"** ในแต่ละ task | "ทุก Task มีบอกว่ามาจากบรรทัดไหนใน PRD" |
| 3 | อ่านตัวอย่าง 1-2 task พร้อม ← มาจาก: "..." | "เช่น Task นี้ — มาจาก Requirement ข้อนี้โดยตรง" |
| 4 | **เปิดไฟล์ `prd-demo.md`** ขึ้นมาคู่กัน (split screen หรือสลับหน้าต่าง) | "ทีนี้ มาดู PRD ต้นฉบับกันครับ" |
| 5 | ใช้ Cmd+F ค้นหาคำว่า **"Test"** ใน prd-demo.md | "ลองหาคำว่า Test ดูครับ" |
| 6 | **ไม่พบผลลัพธ์** — แสดงให้ผู้ชมเห็น | — |
| 7 | หยุดนิ่ง 2 วินาที | "ไม่มีครับ — PRD ฉบับนี้ไม่ได้ระบุ Testing ไว้เลย" |
| 8 | สลับกลับมาที่ Claude response | "นั่นคือเหตุผลที่ AI ไม่แตก Task Testing ออกมา" |
| 9 | — | "AI ไม่ได้พลาด — ปัญหาอยู่ที่ Input ครับ" |

> 💡 **จุดที่ต้องเน้น:** Traceability ช่วยให้เราค้นหาสาเหตุได้เองโดยไม่ต้องรัน AI ใหม่
