# 🎬 Demo Guide — 4.9 Scenes 3–5
## สาธิต AI Workflow: Requirement → PRD (Design → Build → Test → Iterate)

> **ไฟล์ที่ต้องเตรียมไว้ก่อนถ่าย**
> - `01-design-prompt.md` — Prompt ออกแบบ 5 องค์ประกอบ
> - `02-prd-template.md` — PRD Template ของทีม (แนบพร้อม Design prompt)
> - `03-build-prompt.md` — Prompt สร้าง Instruction
> - `04-test-requirement-input.md` — โน้ต Requirement จากลูกค้าสำหรับทดสอบ
> - `05-iterate-nfr-patch.md` — Instruction patch สำหรับ Iterate (เปิดค้างไว้)
> - Claude Desktop เปิดค้างไว้ พร้อมสร้าง Project ใหม่

---

## ⚡ ภาพรวม Flow ทั้งหมด

```
Scene 3 (Design) → Scene 4 (Build) → Scene 5 (Test) → Scene 5+ (Iterate)
01-design-prompt    03-build-prompt   04-test-input     05-iterate-patch
02-prd-template ↗        ↓                 ↓                   ↓
                   Instruction v1    พบว่าขาด NFR      แก้ Instruction → เทสซ้ำ
```

## Scene 5+ — Iterate (ให้ AI เขียน Instruction ใหม่)

**เป้าหมาย:** แสดงให้เห็นว่า Workflow ไม่ต้องสมบูรณ์รอบแรก — บอก AI ว่าขาดอะไร ให้มันเขียน Instruction ใหม่ให้เลย

> เปิดไฟล์ `05-iterate-nfr-patch.md` ค้างไว้เพื่อ copy prompt

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | **New Conversation** (ไม่ต้องอยู่ใน Project ก็ได้) | "แทนที่จะมานั่งแก้ข้อความ Instruction เอง — ขอถาม AI ตรงๆ เลยครับ" |
| 2 | Copy **Prompt สำหรับ Iterate** จาก `05-iterate-nfr-patch.md` → Paste → Send | "บอก AI ว่า Output ขาด NFR และให้มันเขียน Instruction ใหม่ที่ครอบคลุม NFR ให้" |
| 3 | รอ response → **เลื่อนดู Instruction ใหม่** | — |
| 4 | **ชี้ที่ขั้นตอนที่เพิ่มเข้ามาใน ## วิธีทำงาน** | "AI เพิ่มขั้นตอนวิเคราะห์ NFR ทั้งที่ลูกค้าพูดตรงๆ และโดยนัย" |
| 5 | **ชี้ที่ Section NFR ในโครงสร้าง PRD** | "และเพิ่ม Section Non-Functional Requirements เป็นตารางแบ่งตามประเภท" |
| 6 | **Copy** Instruction ใหม่ทั้งหมด | — |
| 7 | ไปที่ Project "PRD Generator" → **Edit Project Instructions** → Paste → Save | "วาง Instruction ใหม่เข้า Project ครับ" |
| 8 | **New Conversation** → แนบ `04-test-requirement-input.md` → Send | "ทดสอบด้วย Input เดิม — เพื่อดูว่า NFR ออกมาไหม" |
| 9 | รอ response → **เลื่อนหา Non-Functional Requirements section** | — |
| 10 | **ชี้ที่ตาราง NFR** | "ตอนนี้มี Non-Functional Requirements ออกมาแล้วครับ" |
| 11 | **ชี้แถว Performance: email 30 นาที** | "AI ดึงออกมาจากที่ลูกค้าบอกว่า 'แจ้งเตือน 30 นาทีก่อนประชุม'" |
| 12 | **ชี้แถว Security: LDAP/SSO** | "และ SSO ที่ลูกค้าพูดถึงก็กลายเป็น Security requirement" |
| 13 | **ชี้แถว Reliability** | "แถว Reliability นี้น่าสนใจ — ลูกค้าไม่ได้พูดตรงๆ แต่ AI อนุมานได้ว่า ระบบจองต้องป้องกัน double-booking" |
| 14 | หยุดนิ่ง 2 วินาที | "นี่คือ Iterate ครับ — ไม่ต้องมานั่งแก้ข้อความเอง แค่บอก AI ว่าขาดอะไร มันเขียน Instruction ใหม่ให้เลย" |

> 💡 **จุดที่ต้องเน้น:**
> "Workflow ไม่มีรอบเดียวสมบูรณ์ — Design → Build → Test → Iterate คือ cycle ปกติ
> สิ่งสำคัญคือเราไม่ต้องเริ่มใหม่ทั้งหมด แค่รู้ว่าขาดอะไร แล้วให้ AI ช่วยเขียน Instruction ใหม่ได้เลย"
