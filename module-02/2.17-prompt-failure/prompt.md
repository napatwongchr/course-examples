คุณเป็นผู้เชี่ยวชาญด้าน Prompt Engineering

วิเคราะห์ prompt ที่ฉันให้มาตาม 4 ปัจจัยต่อไปนี้ แล้วสรุปผลพร้อมข้อเสนอแนะ:

**ปัจจัยที่ 1 — คำสั่งชัดเจนหรือไม่**
ตรวจสอบว่า prompt ใช้คำกำกวมไหม, ระบุรูปแบบผลลัพธ์ที่ต้องการไหม, บอก scope ของงานชัดเจนไหม

**ปัจจัยที่ 2 — ข้อมูลเพียงพอหรือไม่**
ตรวจสอบว่ามีข้อมูลบริบทเพียงพอไหม เช่น โค้ดที่เกี่ยวข้อง, error message, สถานการณ์จริง หรือ AI ต้องเดาเยอะเกินไป

**ปัจจัยที่ 3 — ข้อมูลเกินความจำเป็นหรือไม่**
ตรวจสอบว่ามีข้อมูลที่ไม่เกี่ยวข้องหรือมากเกินไปจนทำให้ AI ไม่รู้จะโฟกัสที่ไหน

**ปัจจัยที่ 4 — บริบทเหมาะสมหรือไม่**
ตรวจสอบว่าระบุ role, framework, constraint หรือ scenario ที่จำเป็นครบไหม เช่น ภาษาโปรแกรม, framework ที่ใช้, ระดับ coverage ที่ต้องการ
