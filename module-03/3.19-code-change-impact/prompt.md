# impact assessment

วิเคราะห์ผลกระทบถ้าจะ refactor getNotesByTopicId()

Impact Assessment แบ่งเป็น 3 ระดับ:

- Direct Impact: ไฟล์ที่ต้องแก้ทันที
- Indirect Impact: ไฟล์ที่ได้รับผลกระทบต่อเนื่อง
- Cascading Impact: ส่วนที่อาจพังโดยไม่คาดคิด
  พร้อม risk level (Low/Medium/High) และ checklist สิ่งที่ต้องทำก่อน merge
