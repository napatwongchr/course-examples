# ✅ Prompt สร้าง README Documentation — Scene 4

ต่อจาก good.md — หลัง AI อธิบาย codebase แล้ว ให้ AI สร้าง documentation จริงๆ:

```
จากโครงสร้าง PLA project ด้านบน ช่วยสร้าง README.md ที่ครอบคลุม:

1. Project Overview (1 ย่อหน้า)
2. Tech Stack (table format)
3. Getting Started (ขั้นตอน install และรัน)
4. Project Structure (file tree + อธิบาย)
5. Key Features (bullet points)
6. Development Guide — วิธี add feature ใหม่ตามแนวทางของโปรเจกต์

ใช้ภาษาอังกฤษ เป็น markdown format ที่ render สวยงามบน GitHub
```

---

> 💬 เมื่อ AI สร้าง documentation ให้แล้ว อย่า copy วางเข้า GitHub เลยทันทีนะครับ
> ต้อง validate ความถูกต้องด้วยตัวเองก่อนทุกครั้ง:
>
> — ตรวจว่า step ใน Getting Started ทำได้จริงไหม (ลองรันตาม)
> — ตรวจว่า feature ที่ระบุมีอยู่จริงในโค้ดไหม
> — ตรวจว่า tech stack ตรงกับ package.json จริงไหม
>
> AI ดูดีครับ แต่อาจเติม feature ที่ยังไม่ได้สร้าง หรือ step ที่ไม่ครบ
> Validate ก่อน — แล้ว README นี้พร้อม commit ได้เลย
