# 🎬 Demo Guide — 5.19 Scenes 1–3
## สาธิตการสร้าง Integration Test ด้วย Claude Code

> **ไฟล์ที่ต้องเตรียมไว้ก่อนถ่าย**
> - `00-it-test-case-design-prompt.md` — Prompt ออกแบบ Integration Test Case
> - `00b-it-test-data-design-prompt.md` — Prompt ออกแบบ Test Data
> - `01-integration-test-case-data.md` — Test Case + Test Data สำเร็จรูป (ผลลัพธ์จาก Scene 1–2)
> - `02-integration-test-prompt.md` — Prompt สั่ง Claude Code สร้าง Integration Test
> - Claude Code เปิดใน Terminal อยู่แล้ว ใน directory ของ PLA Project

## ⚡ ภาพรวม 3 Scenes

```
Scene 1 (Test Case Design) → Scene 2 (Test Data Design) → Scene 3 (Code Generation)
00-it-test-case-design-prompt   00b-it-test-data-design-prompt   02-integration-test-prompt
            ↓                               ↓                              ↓
```

## Scene 1 — Integration Test Case Design

**เป้าหมาย:** ให้ผู้ชมเห็นว่า Integration Test ต่างจาก Unit Test ตรงที่ทดสอบ flow ทั้งระบบ ไม่ใช่แค่ฟังก์ชันเดียว — และ AI ช่วยออกแบบ Cases ที่ครอบคลุมได้

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Claude.ai หรือ Claude Desktop | — |
| 2 | Copy prompt จาก `00-it-test-case-design-prompt.md` → Paste → Enter | "เริ่มจากการออกแบบ Integration Test Case ก่อนเลยครับ" |
| 3 | รอ response → เลื่อนดู IT Cases ที่ได้ | — |
| 4 | **ชี้ที่ flow diagram** ใน prompt (store → validate → score → save → SM-2 → return) | "Integration Test ทดสอบทั้ง flow นี้ใน test เดียว — ไม่ใช่แค่ function เดียว" |
| 5 | **ชี้ที่ Happy Path Cases (IT-01, IT-02, IT-03)** | "Happy Path ทดสอบว่า flow ปกติ ข้อมูลถูก save ลง store จริงๆ" |
| 6 | **ชี้ที่ Error Cases (IT-04, IT-05)** | "Error Case ทดสอบว่าถ้า validation ล้มเหลว — ต้องไม่มีอะไรถูก save เลย" |
| 7 | **ชี้ที่ Schedule Progression (IT-06)** | "Submit ถูก 3 รอบติดกัน — interval ต้องเพิ่มตาม SM-2 algorithm จริงๆ" |
| 8 | **ชี้ที่ Store Integrity (IT-07)** | "2 topic ต่างกัน — schedule ต้องไม่ปนกัน — ทดสอบ store isolation ครับ" |
| 9 | หยุดนิ่ง 2 วินาที | "Unit Test บอกว่า function แต่ละตัวทำงาน — Integration Test บอกว่าทั้ง flow ทำงานร่วมกันได้ครับ" |

## Scene 2 — Integration Test Data Design

**เป้าหมาย:** ให้ผู้ชมเห็นว่า Test Data ของ Integration Test ต้องระบุทั้ง input answers, quiz setup, และ expected store state ที่ตรวจสอบได้จริง

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | **ใน Claude.ai conversation เดิม** ต่อจาก Scene 1 | — |
| 2 | Copy prompt จาก `00b-it-test-data-design-prompt.md` → Paste → Enter | "ต่อด้วย Test Data สำหรับแต่ละ IT Case ครับ" |
| 3 | รอ response → เลื่อนดู Test Data ที่ได้ | — |
| 4 | **ชี้ที่ Answers column ของ IT-01** | "IT-01 ใส่ `[0,1,2,3,0]` — ตรงกับ correctAnswers ทุกข้อ (i%4) — คาดว่า score = 100" |
| 5 | **ชี้ที่ Expected Store State ของ IT-04** | "IT-04 answers น้อยกว่า questions — คาดว่า attempts ว่างเปล่า — ไม่มีอะไร save เลย" |
| 6 | **ชี้ที่ Isolation Note (randomUUID)** | "แต่ละ test ใช้ UUID ใหม่ — ป้องกัน data ปนกันระหว่าง test cases" |
| 7 | หยุดนิ่ง 2 วินาที | "Test Data ที่ดีบอกชัดว่า store ต้องอยู่ในสถานะไหน — ทำให้เขียน assertion ได้ทันทีครับ" |

## Scene 3 — Code Generation

**เป้าหมาย:** ให้ผู้ชมเห็นว่า Integration Test ทดสอบการทำงานร่วมกันของหลายฟังก์ชัน และ Claude Code สร้างได้พร้อมใช้

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Terminal → เข้า directory PLA project | — |
| 2 | รัน `claude` เพื่อเปิด Claude Code | — |
| 3 | **แสดง Test Case** จาก `01-integration-test-case-data.md` | "Test Case ที่ออกแบบไว้ ทดสอบ Flow ตั้งแต่รับ Input ถึงบันทึก store ครับ" |
| 4 | Copy prompt จาก `02-integration-test-prompt.md` → Paste → Enter | "สั่ง Claude Code สร้าง Integration Test ครับ" |
| 5 | รอ Claude Code สร้างไฟล์ test | — |
| 6 | **ชี้ที่ไฟล์ที่สร้าง** `__tests__/quiz-service.integration.test.ts` | "Claude Code สร้างไฟล์ Integration Test ให้เลยครับ" |
| 7 | **ชี้ที่ `seedTopicWithNotes()` helper** | "ใช้ randomUUID() สร้าง topic id ใหม่ต่อ test — ไม่มี data ปน" |
| 8 | **ชี้ที่ IT-01 test** | "ทดสอบว่าหลัง submit แล้ว record ปรากฏใน store จริงๆ — ไม่ใช่แค่ mock" |
| 9 | **ชี้ที่ IT-03 schedule reset test** | "ถ้า score < 60 — repetitionInterval ต้อง reset เป็น 1 เสมอ" |
| 10 | ออกจาก Claude Code → รัน `npx vitest run` | "รัน Integration Test ดูผลครับ" |
| 11 | **ชี้ที่ผลลัพธ์** | "ผ่าน [X] เคส — การทำงานร่วมกันระหว่าง functions และ store ทำงานถูกต้องครับ" |
| 12 | หยุดนิ่ง 2 วินาที | "เริ่มจากออกแบบ → Test Data → สร้างโค้ด — flow ที่ทำให้ test มีคุณภาพจริงๆ ครับ" |
