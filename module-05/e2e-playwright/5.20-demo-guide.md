# 🎬 Demo Guide — 5.20 Scenes 1–3
## สาธิตการสร้าง E2E Test ด้วย Playwright และ Claude Code

> **ไฟล์ที่ต้องเตรียมไว้ก่อนถ่าย**
> - `00-e2e-test-case-design-prompt.md` — Prompt ออกแบบ E2E Test Case
> - `00b-e2e-test-data-design-prompt.md` — Prompt ออกแบบ Test Data + Selectors
> - `01-e2e-test-case-data.md` — Test Case + Test Data สำเร็จรูป (ผลลัพธ์จาก Scene 1–2)
> - `02-e2e-test-prompt.md` — Prompt สั่ง Claude Code สร้าง Playwright Test
> - Claude Code เปิดใน Terminal อยู่แล้ว
> - PLA App รันอยู่บน localhost:3000 แล้ว

---

## ⚡ ภาพรวม 3 Scenes

```
Scene 1 (Test Case Design) → Scene 2 (Test Data Design) → Scene 3 (Code Generation)
```

## Scene 1 — E2E Test Case Design

**เป้าหมาย:** ให้ผู้ชมเห็นว่า E2E Test คิดจากมุม User ไม่ใช่มุม Code — AI ช่วยออกแบบ Cases ที่ครอบคลุม User Journey ได้

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Claude.ai หรือ Claude Desktop | — |
| 2 | Copy prompt จาก `00-e2e-test-case-design-prompt.md` → Paste → Enter | "เริ่มออกแบบ E2E Test Case ในมุม User ก่อนเลยครับ" |
| 3 | รอ response → เลื่อนดู E2E Cases ที่ได้ | — |
| 4 | **ชี้ที่ User Steps ของ E2E-01** | "E2E Test ระบุ steps เหมือน User ทำจริง — เปิดหน้า กดปุ่ม กรอกข้อมูล เห็นผลลัพธ์" |
| 5 | **ชี้ที่ Happy Path Cases (E2E-01, E2E-03)** | "Happy Path ทดสอบว่า user สร้างและลบ topic ได้สำเร็จ — เห็นผลบนหน้าจอ" |
| 6 | **ชี้ที่ Edge Cases (E2E-02, E2E-04)** | "Edge Case ทดสอบว่าถ้า input ว่าง dialog ไม่ปิด — และถ้าไม่มี topic เห็น empty state" |
| 7 | **เปรียบเทียบกับ Unit/Integration** | "Unit Test เช็ค return value — Integration Test เช็ค store — E2E Test เช็คสิ่งที่เห็นบนหน้าจอจริงๆ ครับ" |
| 8 | หยุดนิ่ง 2 วินาที | "E2E Test ใกล้เคียงกับ User จริงที่สุด — ถ้าผ่าน แปลว่า User ใช้งานได้จริงครับ" |

## Scene 2 — E2E Test Data Design

**เป้าหมาย:** ให้ผู้ชมเห็นว่า Test Data ของ E2E ต้องระบุทั้ง selector, ข้อมูลที่กรอก, และ expected UI assertion

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | **ใน Claude.ai conversation เดิม** ต่อจาก Scene 1 | — |
| 2 | Copy prompt จาก `00b-e2e-test-data-design-prompt.md` → Paste → Enter | "ต่อด้วย Test Data — Selectors และ Assertions สำหรับแต่ละ Case ครับ" |
| 3 | รอ response → เลื่อนดู Test Data ที่ได้ | — |
| 4 | **ชี้ที่ Key Selectors column** | "`data-testid` selector — ไม่ใช้ CSS class ไม่ใช้ xpath — เสถียรกว่าเยอะครับ" |
| 5 | **ชี้ที่ Test Data ของ E2E-01** | "title ใช้ `Test Topic ${Date.now()}` — unique ต่อ run — ป้องกัน conflict ระหว่าง tests" |
| 6 | **ชี้ที่ Assertion column** | "Assertion ระบุว่าต้อง expect อะไรบนหน้าจอ — `toBeVisible()`, `not.toBeVisible()` — ชัดเจน" |
| 7 | หยุดนิ่ง 2 วินาที | "Test Data ที่ดีทำให้เราไม่ต้องเดาว่า test ผ่านหรือไม่ผ่าน — มีเกณฑ์ชัดเจนครับ" |

## Scene 3 — Code Generation

**เป้าหมาย:** ให้ผู้ชมเห็นว่า E2E Test ด้วย Playwright ทดสอบในมุมผู้ใช้จริง — ไม่ใช่ code level

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Terminal → เข้า directory PLA project | — |
| 2 | รัน `claude` เพื่อเปิด Claude Code | — |
| 3 | **แสดง E2E Test Case** จากไฟล์ `01-e2e-test-case-data.md` | "Test Case E2E ดูในมุมผู้ใช้ครับ — ตั้งแต่เปิดหน้าจนถึงเห็นผลลัพธ์บนหน้าจอ" |
| 4 | Copy prompt จาก `02-e2e-test-prompt.md` → Paste → Enter | "สั่ง Claude Code เขียน E2E Test ด้วย Playwright ครับ" |
| 5 | รอ Claude Code สร้างไฟล์ test | — |
| 6 | **ชี้ที่ไฟล์ที่สร้าง** `topicManager.spec.ts` | "Claude Code สร้าง Playwright test file ให้เลยครับ" |
| 7 | **ชี้ที่ `page.goto()`** | "Playwright เปิด browser จริง เข้าหน้าเว็บจริง" |
| 8 | **ชี้ที่ `page.click()` และ `page.fill()`** | "จำลองการกดปุ่มและพิมพ์ข้อความ — เหมือน user จริงๆ เลยครับ" |
| 9 | **ชี้ที่ `expect(page.locator(...))`** | "ตรวจสอบผลลัพธ์บนหน้าจอ — ไม่ใช่ return value ของ function" |
| 10 | ออกจาก Claude Code → รัน `npx playwright test` | "รัน E2E Test ดูผลครับ" |
| 11 | รอผลลัพธ์ปรากฏ (อาจเห็น browser เปิด-ปิด) | "Playwright เปิด Browser ขึ้นมาแล้ว simulate User ครับ" |
| 12 | **ชี้ที่ผลลัพธ์ PASSED** | "ผ่านทั้งหมด — ระบบทำงานได้จริงในมุมผู้ใช้" |
| 13 | หยุดนิ่ง 2 วินาที | "เริ่มจากออกแบบ Test Case → Test Data → สร้างโค้ด — นี่คือ E2E Test ที่มีคุณภาพจริงๆ ครับ" |
