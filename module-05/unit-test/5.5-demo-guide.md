## Scene 2 — Demo

**เป้าหมาย:** ให้ผู้ชมเห็นว่า Claude Code เขียน Unit Test ได้ตาม Test Case ที่ออกแบบไว้ และรันผ่าน

| # | ทำบนหน้าจอ | พูด |
|---|-----------|-----|
| 1 | เปิด Terminal → เข้า directory PLA project | — |
| 2 | รัน `claude` เพื่อเปิด Claude Code | — |
| 3 | Copy prompt จาก `02-unit-test-prompt.md` → Paste → Enter | "สั่ง Claude Code เขียน Unit Test ตาม Test Case ที่ออกแบบไว้ครับ" |
| 4 | รอ Claude Code สร้างไฟล์ test | — |
| 5 | Claude Code แสดงไฟล์ที่สร้าง → **ชี้ชื่อไฟล์** `__tests__/sm2.test.ts` | "Claude Code สร้างไฟล์ test ให้เลยครับ" |
| 6 | **ชี้ที่โครงสร้าง describe block** | "describe จัดกลุ่ม Test ตาม scenario — pass path / fail path / boundary / easeFactor" |
| 7 | **ชี้ที่ Pass Path test** | "Test Pass Path — consecutive=0 ครั้งแรก ต้อง interval=1, consecutive=1" |
| 8 | **ชี้ที่ Boundary และ EaseFactor tests** | "Boundary Values — score=60 ผ่าน, score=59 fail ทันที — และ EF ต้องไม่ต่ำกว่า 1.3 เลย" |
| 9 | ออกจาก Claude Code → รัน `npx vitest run` ใน Terminal | "รัน test ดูผลครับ" |
| 10 | รอผลลัพธ์ปรากฏ | — |
| 11 | **ชี้ที่ "X tests passed"** | "ผ่าน [X] เคสจากทั้งหมด" |
| 12 | **ถ้ามี test ที่ fail** | "เคสที่ fail คือสัญญาณว่า Implementation ยังไม่ครบ — ไม่ใช่ว่า Test ผิด" |
| 13 | หยุดนิ่ง 2 วินาที | "Test ที่ดีคือ Test ที่เขียนก่อน code จะ perfect — มันคือ safety net ของเราครับ" |
